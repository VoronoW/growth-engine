## Hard Constraints
- Deployable on GitHub Pages
- Frontend only, no required backend runtime
- Responsive, desktop-first
- Avoid cheap landing page style
- Avoid exaggerated cyberpunk visuals
- Strong hierarchy, restrained motion, premium interaction
- Must include a client intake form for the first 30 days onboarding
- Form must be front-end only with configurable integration options:
  - Formspree
  - Web3Forms
  - Google Forms
  - Notion form embed
- Include README with deployment steps
- Include GitHub Actions workflow for GitHub Pages deployment

## Copy Principles
- Sell certainty, not agency labor
- Sell system reconstruction, not ad buying services
- Emphasize data truth, attribution audit, testing framework, SOP handoff
- Use consulting-grade language, not generic marketing buzzwords

## UX Principles
- Build clear narrative flow:
  pain → diagnosis → system solution → 90-day roadmap → deliverables → fit → differentiation → CTA
- Include dashboard-like panels, timeline tabs, KPI modules, diagnostic cards
- Keep motion tasteful and performant
